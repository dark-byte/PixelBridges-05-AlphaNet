ETH RPC API Documentation

[ Base URL: https://localhost:4001/api/eth-rpc ]

This API is provided to support some rpc methods in the exact format specified for alph nodes, which can be found here. This is useful to allow sending requests to alph without having to change anything about the request. However, in general, the custom RPC is recommended. Anything not in this list is not supported. Click on the method to be taken to the documentation for that method, and check the notes section for any potential differences.

eth_getLogs

      
Will never return more than 1000 log entries.

For this reason, you can use pagination options to request the next page. Pagination options params: {"logIndex": "3D", "blockNumber": "6423AC", "transactionIndex": 53} which include parameters from the last log received from the previous request. These three parameters are required for pagination.


      
        curl -X POST --data '{"id":0,"jsonrpc":"2.0","method": "eth_getLogs", "params": []}'
